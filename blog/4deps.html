我用 4 个依赖做了个白板，替代 Excalidraw

别的白板要你注册。这个不用。别的白板要联网。这个不用。别的白板加载 2MB。这个 160KB。

为什么要做这个

事情的起因很简单：有一天我想在电脑上随手画画，打开 Excalidraw，加载了 10 秒没出来。

我看了一眼 Network 面板——它从 CDN 拉了一堆东西：字体、图片库、协作模块……我只是想画个画。

然后我试了 tldraw，差不多。

更让我在意的是：这两个工具在中国大陆的体验都不好。 CDN 被墙、加载慢、偶尔直接白屏。我不是在抱怨它们——它们都是优秀的开源项目。但我想要的是一个打开就能用的东西：

  • 不注册
  • 不联网
  • 不加载 CDN
  • 不问你要不要开启协作

就是打开，画，关掉。

技术选型：做减法

我给自己定了一个硬约束：生产依赖不超过 5 个。

最终只用了 4 个：

依赖用途
react + react-domUI 框架
zustand状态管理
jspdfPDF 导出
perfect-freehand笔画压感

就这些。没有 UI 库，没有图标库，没有路由库，没有动画库。

UI 自己写。 工具栏、侧栏、弹窗、tooltip——全部手写组件。不是因为我不想用 shadcn，而是每个额外依赖都是一个潜在的加载瓶颈和维护负担。

图标用 SVG inline。 20 多个图标，全部内联到组件里，0 个外部请求。

样式用 CSS Modules + CSS 变量。 一套暖色系变量控制全局，切换暗色模式只需要翻转变量值。

Canvas 性能：不卡的秘密

白板的核心是 Canvas。如果画了 200 个元素就开始卡，那这个工具就没意义。

我用了三个关键优化：

1. 离屏 Canvas 缓存

每个元素渲染一次后，缓存到一个离屏 Canvas。后续重绘时直接 drawImage，不需要重新执行所有绘图指令。

// 每个元素有自己的离屏缓存
const offscreen = new OffscreenCanvas(width, height);
const ctx = offscreen.getContext('2d');
// 绘制一次，后续直接贴图
mainCtx.drawImage(offscreen, x, y);

2. rAF 批处理

状态变化时不立即重绘，而是在下一帧统一处理。多个连续的状态更新会被合并成一次重绘。

let pending = false;
function scheduleRedraw() {
  if (pending) return;
  pending = true;
  requestAnimationFrame(() => {
    render();
    pending = false;
  });
}

3. LRU 图片缓存

用户插入的图片会占用内存。我做了一个 LRU 缓存，上限 50 张。超过后最久未使用的图片会被回收，需要时重新加载。

这三个加在一起，200+ 元素的画布依然流畅。

设计：暖色纸纹

我不想做一个"功能性工具"。我希望打开它的那一刻，感觉是温暖的。

  • 颜色： 主色是 #f5f0e8（暖白）和 #c47a5a（焦赭）。不是纯白，不是冷灰，是旧书页的颜色。
  • 纸纹： 用 SVG 噪点做了一层纸纹理叠加在画布上。不影响性能（纯 CSS），但让整个画面有了质感。
  • 圆角： 统一 14px。
  • 间距： 工具按钮 42px，间距 8px。所有尺寸都是 8 的倍数。
  • 动画： 统一缓动曲线 cubic-bezier(0.4, 0, 0.2, 1)，时长 200ms。

是的，我没有协作功能。这是有意为之。协作是一个巨大的功能，它带来 WebSocket、用户系统、冲突解决、服务器成本……我想要的是一个本地工具，就像你的笔记本，不需要连网就能用。

功能：不多不少

6 种笔刷（钢笔、荧光笔、铅笔、书法笔、虚线笔、霓虹笔），9 种工具（选择、画笔、橡皮、平移、矩形、圆形、文字、直线、箭头），6 种导出格式（PNG、JPG、PDF、SVG、Word、JSON）。

框选、对齐辅助线、形状填充、Shift 约束、图片插入、多画布管理、暗色模式、快捷键。

这些都是"一个白板应该有的功能"。没有做花哨的 AI 功能，没有做模板市场，没有做云端同步。

打开就画，画完就走。

离线版

除了在线版，我还提供了离线版：下载一个 zip，解压后双击 index.html 就能用。不需要安装 Node.js，不需要 npm install，不需要构建。

这是我最初就想做到的事情——让工具回到它最简单的形态。

一个 HTML 文件，打开就能用。
